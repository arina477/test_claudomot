# Triage Routing Table

**Purpose.** When a technical error / bug / failure surfaces in any block (Iron Law: orchestrator does NOT fix directly), this table classifies the symptom and emits a **domain tag**. The orchestrator then looks up the domain tag in `command-center/AGENTS.md` to resolve the actual specialist agent. Two-step lookup: symptom → tag (this file) → agent (`AGENTS.md`).

**Why two layers.** Agent names vary per project; symptoms and domain taxonomies are stable. Routing through tags keeps this file project-agnostic at the schema layer and project-tuned at the row layer.

**Schema (every row):**

```
| Symptom (concrete signal)                         | Domain tag        |
|---------------------------------------------------|-------------------|
| <stack-trace pattern, error class, or condition>  | <kebab-case tag>  |
```

The domain tag must match a tag listed in `command-center/AGENTS.md`. If a row's tag is not in `AGENTS.md`, route through `claudomat-brain/setup-tools/agent-creator/agent-creator.md` to install the missing agent before retrying — never substitute silently.

**Iron Law (per `claudomat-brain/DISPATCHER.md`):** the orchestrator NEVER fixes errors directly. Classify here → look up in `AGENTS.md` → spawn the specialist via the `Agent` tool, OR invoke `/investigate` when classification is ambiguous or no row matches.

**File ownership:** project-owned. Generated at `claudomat init` from `claudomat-brain/setup-tools/install.md` § "Triage table generation" using the universal seed below + project-context expansion (`project.yaml: stack.backend / stack.database / stack.frontend / stack.deploy_platform`, plus SDK list / language-stack from `command-center/dev/architecture/_library.md`). Re-generated on `claudomat sync` only when the seed schema bumps. Founder-edits between syncs are preserved.

|---|---|

| CI infrastructure failure (job runner, artifact upload — not test logic) | `ci-infrastructure` |

## Lookup procedure (orchestrator)

1. Capture the error signal (stack trace, exit code, log line, monitor verdict).
2. Match against the symptom column top-to-bottom; first match wins.
3. Read the domain tag.
4. Open `command-center/AGENTS.md`; find the row whose `expertise` or `routing` matches the tag.
5. Spawn the agent via `Agent(subagent_type=<tag>)`.
6. **No-match path:** invoke `/investigate` instead — let the investigator classify, then come back to this table on next iteration. Do NOT spawn a generic agent or attempt the fix directly.
7. **Tag-not-in-AGENTS.md path:** route through `claudomat-brain/setup-tools/agent-creator/agent-creator.md` to install the missing agent. Recurring need for the same tag is the signal to fold it into `tag-template.yaml`.
